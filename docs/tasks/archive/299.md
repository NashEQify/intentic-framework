# Task 299 — Fabrication-Mitigation: Evidence-Pointer-Schema + Engine-Step-Gate

## Problem

Personas (Buddy/MCA/Sub-Agents/Reviewer-Personas) koennen in Skill-Outputs
Source-Behauptungen schreiben ohne den Tool-Aufruf real auszufuehren — und
inline Bash-Bloecke / Output-Zahlen halluzinieren. Reviewer-Personas (chief,
adversary, code-review, etc.) lesen Skill-Outputs und koennen fabricated
Source-Verifikation nicht von realer unterscheiden — fabricated Authority
propagiert in Multi-Pass-Reviews.

Die Klasse trifft **alle 13 Source-Verifikations-Skills** (spec_board,
code_review_board, architecture_coherence_review, sectional_deep_review,
adversary_test_plan, spec_amendment_verification, pre_build_spec_audit,
retroactive_spec_update, frame, council, spec_authoring, spec_update,
impl_plan_review) plus alle Reviewer-Personas die im Output Source-Claims
machen.

## Intent

Orchestrator-neutrale, **mechanische** (kein LLM-Audit) Mitigation. Schema-
Bind macht Hallucination strukturell deterministisch fangbar: Pointer mit
`quote:`-Field gegen `path` mittels `grep -F` — Match → pass, no-match →
block. Engine-Check am Workflow-Step-Boundary garantiert Mitigation
orchestrator-neutral.

## Beschreibung

Pfad E gelockt aus Council-Synthesis 2026-05-04. Drei Schichten:

### Schicht 1 — Schema (orchestrator-neutral, alle Adapter)

- Neues Protocol: `skills/_protocols/evidence-pointer-schema.md`
- Migration: `agents/_protocols/reviewer-base.md` `evidence:`-Pflicht von
  Prosa auf strukturierte Pointer
- Pointer-Format pro Source-Behauptung:
  ```yaml
  evidence:
    - kind: file_range          # file_range | grep_match | dir_listing
      path: <path>
      lines: <range>
      quote: <1-3-zeilig>
  ```
- `schema_version: 0` = legacy, kein WARN (Backward-Compat)

### Schicht 2 — Engine-Check via Workflow-Engine (orchestrator-neutral)

- Neuer Completion-Check-Type `pointer_check` in
  `workflow_engine.py:check_completion()` (~50 LoC additiv)
- `completion.compound` am `--complete`-Boundary fuer Source-Verifikations-Steps
- Mechanik pro Pointer: Existenz-Check + `grep -F "$quote" $path` + Range-Check
- Match → pass, no-match → `on_fail: block`
- Pre-commit Check 13 SOURCE-VERIFICATION als 2nd-Layer-Backstop, WARN-only initial

### Schicht 3 — CC-Adapter-Bonus (optional, nur unter CC, nur Tier-1)

- PostToolUse-Hook `orchestrators/claude-code/hooks/evidence-pointer-check.sh`
  nach Pattern `board-output-check.sh` (~150 LoC Bash + jq)
- Tier-1 (6 Skills): spec_board, code_review_board,
  architecture_coherence_review, sectional_deep_review, adversary_test_plan,
  spec_amendment_verification
- OpenCode/Cursor laufen nur Schicht 1+2 — orchestrator-neutral garantiert

### Reviewer-Persona-Coverage

Pflicht zentral in `agents/_protocols/reviewer-base.md` — ein Edit, alle
Reviewer-Personas erben (chief, board-adversary, code-adversary,
code-review, code-spec-fit, code-spec-drift, code-data, code-security,
code-domain-logic, code-reliability, code-api-contract, code-docs-consumer,
council-member, etc.).

## Acceptance Criteria

- **AC-1** Evidence-Pointer-Schema in `skills/_protocols/evidence-pointer-schema.md`
  spezifiziert: kind-Inventar (file_range, grep_match, dir_listing),
  Pflichtfelder pro kind, Quote-Length-Cap, Schema-Version
- **AC-2** `agents/_protocols/reviewer-base.md` `evidence:`-Pflicht migriert
  auf Pointer-Schema mit Backward-Compat (schema_version: 0 = legacy, kein WARN)
- **AC-3** `workflow_engine.py` neuer Completion-Type `pointer_check`:
  Existenz-Check + Quote-Match-Check (`grep -F`) + Range-Check
- **AC-4** Synthetisches Eval (3 konstruierte Szenarien):
  - Quote-Mismatch → blockiert
  - Range-Overflow → blockiert
  - Valid pointer + falsche Schlussfolgerung → mechanisch NICHT gefangen
    (akzeptiertes Trade-off: Schema faengt Fabrication, nicht Misinterpretation)
- **AC-5** Tier-1-Skill-Migration (6 Skills) auf Pointer-Schema mit
  Frontmatter-Marker fuer Tier-1 (Form: Spec-Phase entscheidet — Frontmatter-
  Field oder mechanisch ableitbar aus invocation.primary)
- **AC-6** CC-PostToolUse-Hook
  `orchestrators/claude-code/hooks/evidence-pointer-check.sh` nach Pattern
  `board-output-check.sh`
- **AC-7** Pre-commit Check 13 SOURCE-VERIFICATION (WARN-only initial)
- **AC-8** Performance: Engine-Check + Hook-Latency unter 10% Mehrkosten
  gegen Baseline-Skill-Run
- **AC-9** Backward-Compat: existing Skill-Outputs ohne Schema bleiben
  gueltig (schema_version: 0 = legacy)
