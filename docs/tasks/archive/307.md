# Task 307: OSS-Cleanup Sweep — strip forensic refs across framework

## Context

The framework is being prepared for OSS public surface. Per the
project-level OSS-readability constraint, framework-canonical files
must not embed forensic references to private/consumer-repo work:
consumer-repo task IDs (e.g. "BuddyAI Task 467"), internal session
numbers ("Session 3"), dated session-handoff refs, commit hashes,
project-specific module names in worked examples
(e.g. `chat.capability_llm`, `audit.contracts.audit`).

A spot sweep on 2026-05-10 found:

- **~10 files with consumer-repo task references** in `agents/`
  + `_protocols/`: code-docs-consumer, code-data, code-review,
  code-security, code-domain-logic, code-adversary, code-api-contract,
  solution-expert, test-skeleton-writer, plus 2 reviewer-protocols.
- **~15 files with Task-NNN / Session-N / dated worked-example
  refs** in `skills/` + `agents/` + `framework/`: task_creation,
  plan-review, evidence-pointer-schema, shipping_and_launch,
  architecture_coherence_review, api_and_interface_design,
  code-review, code-api-contract, plan-adversary, navigation, plus
  multiple REFERENCE.md files.

The first targeted cleanup (`skills/spec_authoring/SKILL.md`) was
done inline during a different work block. This task scopes the
broader sweep.

## Scope

For each affected file:

1. Identify forensic references via grep patterns (consumer-repo
   names, Task-NNN matching consumer-repo numbering ranges,
   session-N, dated session-handoff filenames, commit hashes,
   module-path patterns specific to consumer repos).
2. For each match, decide: replace-with-generic / remove / keep.
   - **Replace-with-generic** when the example is illustrative and
     can be rewritten as a pattern without the consumer-repo
     specifics (e.g. "module X imports symbol Y from module Z").
   - **Remove** when the reference is purely historical and adds no
     pattern value.
   - **Keep** when the reference is to the framework's own public
     task numbering (300+) or to an upstream pattern (Plan Agent,
     TeamCreate, verification-agent).
3. Apply the rewrite. Verify the file still validates against its
   structural checks (consistency_check, frontmatter, SKILL anatomy).

## Workflow

- [ ] Step 1: Comprehensive grep — produce a delta-report listing
      every file × every forensic match × proposed action
      (replace / remove / keep).
- [ ] Step 2: Per-file rewrite, batched by file type
      (agents/ first, skills/ second, framework/ + workflows/
      third).
- [ ] Step 3: Validation — run `consistency_check` and any
      structural validators; ensure no broken cross-refs.
- [ ] Step 4: Spot review — random sample 5 files post-rewrite
      and verify the pattern illustrations still teach the lesson
      they did before, just without the forensic specifics.
