## Context

The framework's `code_review_board` skill currently has two levels:

- **L1** (focused, 2-3 reviewers): default for small-scope work.
- **L2** (full board, 5-7 reviewers): substantial work, schema
  changes, cross-spec, etc.

Even L1 dispatches multiple reviewers in parallel. The upstream
agentic-CLI takes a different approach for verification: a single
`verification-agent` persona (`agents/AgentTool/built-in/verificationAgent.ts`)
with strict anti-rationalization heuristics, evidence-required
output format, and an explicit VERDICT: PASS | FAIL | PARTIAL
contract that ends every check.

For genuinely small-scope changes (single-file fix, ≤2 ACs,
no schema change, no cross-spec impact), the upstream single-
verifier pattern delivers equivalent quality at substantially
lower cost.

This task adds a third path — **light** — that uses a single
verification-agent in place of the multi-reviewer board. L1 and L2
remain as today; light is below L1.

## Three-path table (sketch)

| Path | Trigger | Reviewer composition | Output |
|---|---|---|---|
| **Light** (NEW) | ≤2 ACs AND single-file AND no schema AND no cross-spec AND no new behaviour | Single verification-agent | VERDICT: PASS \| FAIL \| PARTIAL with evidence per check |
| **L1** (existing) | small-scope but above light threshold | 2-3 reviewers + chief | Consolidated verdict |
| **L2** (existing) | substantial: >5 files OR new module OR cross-spec OR schema OR effort=L\|XL | 5-7 reviewers + chief | Consolidated verdict |
