# Navigation: runbooks/

## What lives here?

All workflow runbooks. Each workflow is a sub-directory with
`WORKFLOW.md` as SoT. Workflows are the user-facing layer — the user
triggers a workflow ("solve", "build", "fix", "save"); skills are
called by workflow steps, not directly by the user.

**Routing:** `framework/process-map.md` (which workflow for which
work).
**Index:** `framework/boot-navigation.md` (all workflows + skills on
one page).

**5-phase standard** (specify / prepare / execute / verify / close)
is the convention for the producer class (build / fix / review /
solve). Other classes (documentation / operations / maintenance /
bootstrap) have their own structures.

**Classification:** 8 DISTINCT, 0 ORCHESTRATION / REDUNDANT /
WRONG-ABSTRACTION, 1 REPO-SCOPE-DRIFT (`selfhost/` — user-specific,
belongs in a consumer repo). `project_bootstrap/` was removed
2026-05-01.
