# Navigation: _protocols/

## What lives here?

**Persona-level protocols** — required mechanics inlined into every (or
many) reviewer and council persona. On dispatch Buddy assembles the
prompt as: persona definition + board protocol + reasoning trace +
first-principles check + reviewer base.

**Hierarchy:**
1. `reviewer-base.md` — applies to ALL review agents (Spec / Code /
   UX board).
2. `<board>-reviewer-protocol.md` — board-specific rules (spec / code
   / ux).
3. `reviewer-reasoning-trace.md` + `first-principles-check.md` —
   required sections before review output.
4. `code-reviewer-base-extended.md` — extended conventions for the
   Code Board.

Boundary against `skills/_protocols/`: those are skill-level mechanics
(cross-loaded by skills). The protocols here are persona-level
(cross-loaded on persona dispatch).
