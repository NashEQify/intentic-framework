# Boot Navigation — Workflow + Skill Index

**Purpose:** compact index for Buddy boot. Prevents orphan blindness
(skills that are "invisible" in workflows because Buddy doesn't have
the full landscape in mind).

**SoT:** workflow details live in each runbook
(`workflows/runbooks/<name>/WORKFLOW.md`). Table paths below are from
**repository root**. Skill details (taxonomy, composition, maturity
registry, boundaries) are in `framework/skill-map.md`.
Workflow routing logic is in `framework/process-map.md`.

**Maintenance:** maintained manually. When adding workflows/skills,
keep this file and `skill-map.md` in sync. The `consistency_check`
skill validates boot-map drift (check 6) and navigation-layer drift
(check 8).

**Navigation layer (reverse lookup):** each top-level depth-3 directory
has a `navigation.md` with reader-journey sections (`## What lives here?`,
`## Where to look for which question?`) plus an AUTO block (regenerated by
`scripts/generate_navigation.py`). Entry points:

- `framework/navigation.md` — framework sub-areas
- `skills/navigation.md` — all active skills
- `workflows/runbooks/navigation.md` — all workflows
- `agents/navigation.md` — all personas
- (`skills/_protocols/`, `references/`,
  `agents/buddy/`, `agents/_protocols/` each has its own)

## Utility exception (important mental model)

**Rule:** users call workflows only (or steps within them). Skills are
used by workflow steps, not called directly by users. Skills without a
workflow-step call are orphan candidates.

**Exception:** utility skills (table above) are called directly by Buddy
as orchestrator work, not via workflow steps. That is cross-cutting
infrastructure and by definition not workflow-embedded.
`task_creation`, `task_status_update`, `consistency_check`,
`transparency_header`, `return_summary` are not orphans — they are
utility patterns.

---

## Orphan check (quick reference)

A skill is an orphan when:
1. It is `type: capability` (not utility/protocol).
2. It is called in **no** workflow runbook step (grep in
   `workflows/runbooks/*/WORKFLOW.md`).
3. It has no documented "intentional exception" (e.g. utility skill
   called directly by Buddy).

Orphan candidates need content review: what the skill does, whether the
use case is covered elsewhere, and whether it should be integrated into
a workflow (content-based check, not regex-based).
